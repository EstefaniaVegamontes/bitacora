JavaScript Conceptos Básicos

1. Variables Las variables en JavaScript se utilizan para almacenar datos. Puedes declararlas usando var, let o const: var: Es la forma más antigua de declarar variables. Tiene un alcance de función, lo que significa que está disponible en todo el bloque de función en el que se declara. let: Introducido en ES6, let tiene un alcance de bloque, lo que significa que solo está disponible dentro del bloque donde se declara. const: Similar a let, pero una vez asignado un valor, no puede ser reasignado.

2. Tipos de Datos JavaScript maneja varios tipos de datos, entre los más comunes están: Números (number): Incluyen enteros y decimales. Cadenas de texto (string): Textos encerrados en comillas simples o dobles. Booleanos (boolean): Representan valores de verdad (true o false). Arreglos (array): Colección ordenada de elementos. Objetos (object): Estructuras de datos que permiten almacenar pares clave-valor.

3. Operadores Los operadores permiten realizar operaciones con valores. Algunos ejemplos son: Aritméticos: +, -, *, /, % (suma, resta, multiplicación, división, módulo). De asignación: =, +=, -=, *=, /= (asignar y operar en el mismo paso). De comparación: ==, ===, !=, !==, >, <,>=, <= (comparan valores). Lógicos: &&, ||, ! (y lógico, o lógico, negación).

4. Condicionales Los condicionales permiten ejecutar diferentes bloques de código según si una condición es verdadera o falsa. Los más comunes son if, else if, y else.

DOM

El DOM (Document Object Model) es un concepto clave en el desarrollo web, ya que permite la manipulación dinámica de documentos HTML y XML mediante lenguajes de programación como JavaScript. El DOM convierte la estructura de un documento en un modelo o representación en forma de árbol, donde cada elemento del documento (como etiquetas HTML, atributos, textos, etc.) se convierte en un nodo que puede ser accedido y manipulado.

Tipos de Nodos en el DOM Element Nodes: Representan las etiquetas HTML. Por ejemplo, h1, p, etc. Text Nodes: Contienen el texto dentro de un elemento. Por ejemplo, el contenido de un h1 es un nodo de texto. Attribute Nodes: Representan los atributos de las etiquetas HTML. Aunque no son nodos hijos, puedes acceder a ellos mediante los nodos de elementos. Document Node: Representa el documento completo, el cual es la raíz del árbol DOM.

Referenciar con JS

Referenciar elementos con JavaScript se refiere al proceso de acceder y manipular los elementos de una página web a través del DOM (Document Object Model). Esto es fundamental para poder modificar dinámicamente el contenido, estilos y comportamiento de una página web.

Cómo Referenciar Elementos en el DOM Para referenciar un elemento en el DOM, puedes utilizar varios métodos que te permiten seleccionar un elemento específico o un grupo de elementos. Una vez referenciados, puedes manipularlos directamente usando JavaScript.

1. getElementById(id) Este método selecciona un elemento por su atributo id. Es útil cuando sabes que el elemento tiene un identificador único.

2. getElementsByClassName(class) Este método selecciona todos los elementos que tienen una clase específica. Devuelve una colección de elementos, por lo que necesitarás iterar sobre ellos si deseas manipular varios elementos a la vez.

3. getElementsByTagName(tag) Este método selecciona todos los elementos que tienen una etiqueta específica, como div, p, h1, etc. También devuelve una colección de elementos.

Propiedad Style

La propiedad style en JavaScript se utiliza para acceder y modificar los estilos CSS de un elemento HTML directamente desde el script. Esta propiedad es especialmente útil cuando deseas aplicar estilos dinámicamente o cambiar la apariencia de un elemento en respuesta a acciones del usuario.

La propiedad style te permite acceder a los estilos en línea de un elemento, es decir, los estilos que se aplican directamente al atributo style del HTML. No te permite acceder a los estilos definidos en hojas de estilo externas o en bloques style dentro del HTML.

Limitaciones de style Solo afecta los estilos en línea: La propiedad style solo se refiere a los estilos en línea del elemento, lo que significa que no puedes usarla para acceder a los estilos definidos en una hoja de estilo externa. Si un estilo se define en un archivo CSS externo y no está presente en el atributo style del elemento, no podrás acceder a él usando style. No reemplaza las clases: Aunque style es útil para cambios específicos, si necesitas aplicar muchos estilos a la vez, es mejor usar clases CSS y cambiarlas dinámicamente con classList.

Classlist

La propiedad classList en JavaScript es una forma conveniente de trabajar con las clases CSS de un elemento. Te permite añadir, quitar, alternar y verificar las clases de un elemento sin necesidad de manipular directamente el atributo class. Esto es útil para aplicar estilos y comportamientos dinámicos en una página web.

1. add() Este método añade una o más clases a un elemento. Si la clase ya está presente, no la añade de nuevo.

2. remove() Este método elimina una o más clases de un elemento. Si la clase no está presente, no hace nada.

3. toggle() Este método alterna una clase en un elemento. Si la clase está presente, la elimina; si no está presente, la añade.

4. contains() Este método verifica si un elemento tiene una clase específica. Devuelve true si la clase está presente y false si no lo está.

Eventos

En JavaScript, los eventos son acciones que ocurren en la página web y a las que puedes responder usando código. Los eventos permiten que las páginas web sean interactivas, reaccionando a las acciones del usuario como hacer clic en un botón, mover el ratón, escribir en un campo de texto, y más.

Eventos de Mouse:

  • click: Ocurre cuando el usuario hace clic en un elemento.
  • dblclick: Ocurre cuando el usuario hace doble clic en un elemento.
  • mouseover: Ocurre cuando el usuario mueve el ratón sobre un elemento.

    • Eventos de Formulario:

  • submit: Ocurre cuando se envía un formulario.
  • change: Ocurre cuando el valor de un elemento de formulario cambia (como un input o select).
  • focus: Ocurre cuando un elemento recibe el enfoque (por ejemplo, cuando el usuario hace clic en un campo de texto).

    • Eventos de Teclado:

  • keydown: Ocurre cuando una tecla es presionada.
  • keyup: Ocurre cuando una tecla es soltada.
  • keypress: Ocurre cuando una tecla es presionada y soltada (aunque está siendo reemplazado por keydown y keyup en muchos casos).

    Viewport

    El viewport es el área visible de una página web en la pantalla de un dispositivo. En términos más simples, es la parte de la página que el usuario puede ver sin hacer scroll. El concepto de viewport es crucial para el diseño web adaptable (responsive design), ya que permite que las páginas web se ajusten correctamente a diferentes tamaños de pantalla, desde dispositivos móviles hasta pantallas de escritorio.

    Propiedad viewport en HTML

    Para controlar cómo se comporta el viewport en dispositivos móviles y ajustar el diseño según el tamaño de la pantalla, se utiliza una etiqueta especial en el HTML, llamada meta viewport. Esta etiqueta le indica al navegador cómo escalar y dimensionar la página.

    meta name="viewport" content="width=device-width, initial-scale=1.0"

    width=device-width: Ajusta el ancho del viewport al ancho de la pantalla del dispositivo. Esto asegura que la página se muestre correctamente en diferentes dispositivos, desde teléfonos móviles hasta pantallas de escritorio.

    initial-scale=1.0: Establece el nivel de zoom inicial cuando se carga la página. Un valor de 1.0 significa que no habrá zoom, y la página se mostrará al 100% de su tamaño real.

    Importancia del Viewport en Diseño Adaptable

    El viewport es fundamental para crear diseños adaptables, donde el contenido de la página se ajusta dinámicamente a diferentes tamaños de pantalla. Sin la etiqueta viewport, las páginas web a menudo se verían demasiado pequeñas en pantallas móviles, ya que el navegador intentaría mostrar toda la página web como lo haría en una pantalla de escritorio.

    @ Medias

    Las media queries o consultas de medios (utilizando @media en CSS) son una característica clave del diseño web adaptable (responsive design). Permiten aplicar estilos CSS específicos dependiendo de las características del dispositivo o del tamaño de la pantalla del usuario. Esto es esencial para que tu página web se vea bien y funcione correctamente en una variedad de dispositivos, desde teléfonos móviles hasta pantallas de escritorio grandes.

    ¿Qué es @media?

    La regla @media en CSS se utiliza para establecer condiciones que deben cumplirse para que se apliquen ciertos estilos. Estas condiciones pueden basarse en el ancho o altura de la pantalla, la resolución del dispositivo, la orientación (horizontal o vertical), y otros aspectos.

    Tipos de Condiciones Comunes en Media Queries

    max-width: Aplica estilos cuando el ancho del viewport es menor o igual a un valor específico.

    min-width: Aplica estilos cuando el ancho del viewport es mayor o igual a un valor específico.

    max-height: Aplica estilos cuando la altura del viewport es menor o igual a un valor específico.

    min-height: Aplica estilos cuando la altura del viewport es mayor o igual a un valor específico.
